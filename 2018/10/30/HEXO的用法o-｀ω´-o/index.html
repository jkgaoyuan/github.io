如何创建文章

hexo new [layout] <title>

eg：hexo new “example”

layout 系统默认为 post

要注意的是 如果将 layout设定为 drafts 则默认不会在页面中显示，除非添加参数 --draft 或者设置render——drafts 参数设定为true
提示：换行两个空格 下一段一个回车

布局layout

有三个 post、page和draf属性
这三个属性分别对应不同的路径
post source/_posts
page source
draft source/_drafts
我们可以在_config.yml 中修改 default_layout的参数来修改默认属性

草稿draft

草稿默认不会显示在页面上，但是我们可以在执行时加上 –draft，或者将render_drafts 参数设置为 true 来预览。
hexo new draft <title> //新建草稿
eg: hexo new draft "example"
hexo publish [layout] <title> //将草稿移动到文章目录
eg: hexo publish "example"

模版

hexo new 'postname' //新建文章
hexo 会在 scaffolds文件夹中去寻找post.md文件作为模版来新建文章。我们也可通过修改post.md 文件来自定义自己的模版，以下命令同上。
hexo new page "pageName" //新建页面
hexo new draft <title> //新建草稿

Front-matter

每个通过hexo创建的md文件都有的一个区域。可以指定文件的变量。

categories:
- Diary
tags:
- PS3
- Games
需要注意的是 hexo的分类是不可以指定多个同级别的分类的。tags则没有这个问题。

eg:
categories:
- Diary
- Life

这样写并不表示这篇文章会被分别分类到 Diary 和 Life 中 而是将其分类到 Diary下的Life中。也就是说这是有从属关系的。

以下内容是写在Front-matter中的。

title: 使用Hexo搭建个人博客
layout: post
date: 2014-03-03 19:07:43
comments: true （在安装插件后使用-参考）
categories: Blog
tags: [Hexo]
keywords: Hexo, Blog （seo优化）
description: 生命在于折腾，又把博客折腾到Hexo了。给Hexo点赞。
//文章描述信息(对于文章的描述，会直接显示在文章的最前端和阅读文章时的头部)

部署

hexo clean //清除缓存,网页正常时可以忽略

hexo generate = hexo g //生成静态页面止public目录

hexo server = hexo s //本地预览访问

hexo deploy = hexo d //将本地文件推送到远端服务器
